## Discoveries / bugs (loop appends here)
- 2026-06-11: `inngest-cli` downloads its binary in `postinstall`; pnpm 10 blocks it unless `inngest-cli` is listed under `onlyBuiltDependencies` in `pnpm-workspace.yaml`, followed by `pnpm install`/rebuild. `@inngest/test` can assert a step's output reliably with `executeStep(stepId, ...)`; don't rely on final-run `state[stepId]` using the human step id.
- 2026-06-11: vitest runs test files in parallel processes — concurrent `migrate()` calls race on UNAPPLIED migrations (duplicate CREATE TYPE/TABLE; latent until a new migration lands). All DB test files must use `migrateSerialized()` from `src/db/test-support.ts` (pg advisory lock), never `migrate()` directly.
- 2026-06-11: ubs's "secret compared with ==/!=" checker strips comments BEFORE honoring `ubs:ignore`, so inline suppression cannot work for that one checker — restructure instead (avoid `==`/`!=` in statements whose identifiers smell secret-ish; a `switch` or truthiness check passes). Other checkers honor `ubs:ignore` normally.
- 2026-06-11: `next build` evaluates route-module scope with NODE_ENV=production while collecting page data — never call `getEnv()`/`getAuth()` at module scope in routes (prod-only env requirements would fail the build); resolve per-request (`toNextJsHandler((req) => getAuth().handler(req))`).
- 2026-06-11: drizzle-kit emits `ADD COLUMN ... NOT NULL` without a default for new required columns — fails on non-empty tables; hand-edit the migration to add-nullable→backfill→SET NOT NULL (snapshot only records the final state, so future generates don't fight it).
- 2026-06-11: Better Auth drizzle adapter resolves fields by the drizzle table's TS property names; `user.fields.name → "displayName"` remaps core fields; `advanced.database.generateId: false` lets Postgres `gen_random_uuid()` own ids; org-plugin role strings are written verbatim to `members.role`, so the `league_role` pg enum doubles as validation (role keys in `src/auth/permissions.ts` must match it exactly).
