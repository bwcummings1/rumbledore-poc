# T8 Handoff - Data Book Save/Push Controls + Finalized Mode

## What T8 Built
- Extended the existing `/leagues/[leagueId]/data` Data Book view; no parallel page or mid-page control card was added.
- Added steward-only controls in the Data Book masthead:
  - Save checkpoint -> `POST /api/leagues/[leagueId]/curation/checkpoints`
  - Restore checkpoint -> `POST /api/leagues/[leagueId]/curation/checkpoints/[checkpointId]/restore`
  - Push selected season -> `POST /api/leagues/[leagueId]/curation/push` with `{ action: "push", season, checkpointId? }`
  - Push all -> `POST /api/leagues/[leagueId]/curation/push` with `{ action: "pushAll", checkpointId? }`
  - Mark finalized / return live -> new season-mode route
- Added per-season and overall indicators for:
  - unsaved draft edits
  - saved-but-unpushed checkpoints
  - pushed canonical snapshots
- Added push confirmation dialogs that state the canonical boundary and T4 composition invariant.
- Non-stewards keep the read-only Data Book and state indicators, but see no save/restore/push/finalize controls.

## New Persistence
- New RLS-scoped table: `league_curation_season_states`
  - one row per `(league_id, season)`
  - `mode` is `"live"` or `"finalized"`
  - `finalized_at`, `finalized_by_user_id`, and `reason` record the explicit owner/steward action
  - migration: `src/db/migrations/0054_data_book_save_push_state.sql`
- Service exports from `src/stats/index.ts`:
  - `setCurationSeasonMode`
  - `listCurationSeasonStates`
- API route:
  - `POST /api/leagues/[leagueId]/curation/seasons/[season]/mode`
  - body: `{ "mode": "finalized" | "live", "reason"?: string }`
  - requires `minRole: "data_steward"`

## State Model
- The active saved baseline is the latest checkpoint save marker or restore marker in `league_data_edits`, not simply
  the newest checkpoint row. This keeps restore semantics correct.
- Unsaved draft = a data mutation marker after the active save/restore marker.
- Saved-unpushed = the active checkpoint covers a season and that season's latest push is from a different checkpoint
  or does not exist.
- Pushed canonical = a latest push row exists for that season, even if it is now stale behind a newer saved draft.
- Live seasons display as provider-streamed and are not pushable from the UI.
- Finalized seasons display as curate-and-push and are locked to the latest pushed snapshot until pushed again.
- Provider-complete auto-suggestion appears when the selected season is still live and either:
  - it is older than the league's current season, or
  - it is the league's current season and `leagues.status === "complete"`.

## Gotchas
- `pushCurationSeason` still promotes the latest saved checkpoint when no `checkpointId` is sent. The UI sends the
  active checkpoint id when available so restore state is explicit.
- Push-all is disabled in the UI while any season is still live; the T4 API still pushes every season in the active
  checkpoint when called.
- Restore mutates draft tables and then refreshes the route; it does not rewrite checkpoint/push history.
- The new season-mode table is stateful/upserted, not append-only. It is RLS-scoped and FORCE RLS-enabled.
