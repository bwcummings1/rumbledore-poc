# T9 Handoff - Record Book Pushed Snapshot Projection

## What T9 Built
- Re-pointed the Record Book read path to `composeCanonicalSnapshot(db, { leagueId })`.
- Records now derive from the union of each season's latest pushed canonical snapshot:
  - pushed `persons`
  - pushed `teamSeasons`
  - pushed `identityMappings`
  - pushed `weeklyStatistics`
  - pushed `seasonSettings`
  - pushed `groupings`
- Removed live/materialized fact reads from the Record Book loader. Saved checkpoints and draft edits remain invisible
  until a season is pushed.
- Added the explicit no-pushed-data state in `/records`: "No pushed data yet — push from the Data Book."
- Preserved the existing Records UI/components. No parallel records view was added.

## Reopened Follow-Up
- Fixed React duplicate-key warnings in the Record Book render path after the real 95050 vertical slice exposed
  repeated labels/team display strings in lists.
- Main Record Book compact lists now carry stable row IDs built from record type, person IDs, matchup IDs, season,
  and scoring period rather than deriving keys from labels such as "Highest combined" or display names.
- Current-record cards and manager-held record cards use record type plus holder/opponent/season/week/value identity
  with the map index only as a final disambiguator.
- Manager weekly lists, placement role chips, head-to-head meeting cards, rivalry lists, and keeper milestone cards
  were also hardened against repeated names/labels.
- No data-shape or display-rule change: the UI still shows latest pushed team name plus real name.

## Display Rule
- The Record Book now has one display row per person.
- The display name is:
  - latest pushed team name for that person
  - plus the person's canonical real name
  - rendered as `Team Name (Real Name)` when a team name exists
- This collapses serial-renamer variance across years without splitting one person into per-season records.

## Lens / Era Behavior
- Segment pills remain view controls over the already-computed pushed records.
- Era/grouping pills are built only from confirmed groupings present in the pushed canonical snapshot.
- The Record Book does not define eras and does not read live grouping edits. A saved grouping edit must be pushed
  with its season before the lens sees it.

## Gotchas
- `recomputeLeagueStatistics` still exists for live/materialized stats and ingestion-era aggregates. The Record Book
  page loader is the boundary that now derives the displayed catalog from pushed snapshots.
- `composeCanonicalSnapshot` returns repeated person snapshots across pushed seasons. The Record Book must continue
  deduping by person id and choosing latest pushed team-season display.
- Do not reintroduce live `season_statistics`, `weekly_statistics`, `all_time_records`, or live grouping reads into
  the Records loader. Data Book is the draft/live surface; Record Book is pushed-only.
- A league with materialized stats but no pushes is intentionally empty in Records.
- The real 95050 screenshot spec assumes `scripts/verify-records-pushed-snapshot.ts` has already populated and pushed
  the local database.
