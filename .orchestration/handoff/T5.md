## Read Sources
T5 reads the live draft/substrate tables directly inside `withLeagueContext()` with explicit `league_id` filters:
- `persons`
- `identity_mapping`
- `team_season`
- `league_season_settings`
- `season_statistics`
- `weekly_statistics`
- `fantasy_matchups`

Important: T5 does **not** read `composeCanonicalSnapshot`. That remains T9's record-book input after save/push is wired.

## Grain Behavior
- People:
  - One row per `team_season` for the selected season.
  - Shows the mapped `persons.canonical_name`, that season's `team_season.team_name`, owner source names, provider
    team id, mapping method, and confidence.
- Settings:
  - Key-value table of `league_season_settings` fields plus season totals.
  - Includes league size, regular/matchup/championship periods, playoff teams, playoff matchup length, lineup slots,
    scoring type, acquisition, keeper/dynasty flags, W-L-T totals, PF totals, team-week facts, matchup facts, and bye facts.
- Weeks:
  - One row per `weekly_statistics` team-week fact.
  - Shows score, opponent, result, span, playoff/championship segment, and PA.
  - Bye rows render as one-sided (`opponent = BYE`) and preserve `scoring_period_span`.

## UI Decisions
- `PublicationMasthead` now accepts an optional `controls` slot. It renders above the tab row, so the section nav
  remains at the bottom of the top card.
- `TabLinks` now supports the Data Book in button mode via the already-existing `TabButtonItem` path.
- `tabClassName` now includes `min-h-11` to make tabs explicit 44px targets.
- The Data Book tables use `DataTable`, which provides desktop tables and mobile card fallback. No mid-page button
  card or parallel tab component was added.
- Empty/clean seasons show plain DataTable empty rows, with no extra warning chrome.

## Gotchas
- The shell nav destination id is `data`; the route segment is also `data`.
- The year picker currently keeps state client-side. It does not write query params.
- The Data Book read model intentionally loads all seasons into the client view for fast tab/year switching. Current
  95050 volume is small enough for this read view; revisit if future providers bring much larger league histories.
- Mobile duplicate text in tests is expected because `DataTable` renders desktop table and mobile card markup in the DOM.
- T8 save/push controls should stay separate from this T5 read-only surface until the save/push task explicitly wires them.
- T9 must continue to treat this route as live draft data and only re-point Records to `composeCanonicalSnapshot`.
