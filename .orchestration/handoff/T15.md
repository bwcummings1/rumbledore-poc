# T15 Handoff - Complete ESPN Decoding

## What landed

- Added the ESPN decoding source of truth:
  - `src/providers/espn/reference-data.ts`
  - Complete cwendt-derived maps for position/default-position ids, lineup/eligible-slot ids, pro teams, ACTIVITY_MAP
    transaction categories, and scoring stat categories/keys.
  - Includes cwendt's blank id `22` sentinel as canonical `N/A` because real ESPN `eligibleSlots` payloads expose it.
- Added the provider boundary:
  - `src/providers/decoding.ts`
  - ESPN is wired now; Sleeper/Yahoo can plug in by adding provider dictionaries.
- ESPN normalization now decodes through the shared dictionary:
  - Player `defaultPositionId`, `eligibleSlots`, and `proTeamId`.
  - Roster and draft `lineupSlotId` metadata.
  - Scoring settings persist cheap decoded metadata: `providerStatId`, `statCategory`, `statKey`.
  - Transaction numeric/string activity values categorize as add/drop/waiver/trade.
- Data Book Settings no longer has a local partial lineup-slot label map; it formats slot counts through the shared ESPN dictionary while preserving raw numeric ids for settings signatures.

## Integrity

- New `data_integrity_check_key`: `provider_code_decoding`.
- Migration: `src/db/migrations/0059_provider_code_decoding.sql` plus `_journal.json`.
- The invariant scans observed provider codes from:
  - `fantasy_players.metadata.defaultPositionId`, `proTeamId`, `eligibleSlots`
  - `fantasy_roster_entries.metadata.lineupSlotId`
  - `fantasy_draft_picks.metadata.lineupSlotId`
  - `league_season_settings.lineup_slot_counts` and `scoring_settings.scoringItems[].statId`
  - `fantasy_transactions.details.rawActivityTypeId`, raw numeric type values, and item type values
- It flags any undecoded position/slot/proTeam/scoring-stat/activity id and uses the existing stale pass/fail replacement behavior on rerun.
