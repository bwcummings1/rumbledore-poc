# T2 Handoff - Names Ingestion Fix + Clean DB Isolation

## What T2 Built
- Fixed stale/person fixture names by refreshing non-manual `persons.canonical_name` from provider owner names during identity resolution.
- Made targeted current-sync recompute run identity resolution even when there are no changed finalized matchups.
- Added a scoped import-summary helper used by the real-league harness.
- Updated `scripts/import-real-league.ts` to reset only ESPN 95050, run current + 15-season historical import, run full stats recompute, and write a league-scoped `Persons` section.

## Root Cause
- `persons.canonical_name` was written only at person creation time from `team_season.owner_names`.
- If earlier data had created person rows with fixture-like names, later real `fantasy_members.display_name` updates changed `team_season.owner_names` but did not rename the existing person.
- `recomputeChangedMatchupStatistics` returned immediately when `matchupIds` was empty, so a current-member-only sync could report members `0/N changed` and skip identity refresh entirely.
- The T1 summary did not list persons, so it could not prove whether verification was reading the imported real league or fixture league rows.
