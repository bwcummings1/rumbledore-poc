# T13 Handoff - Import Clean Guarantee

## Root Cause
- Real ESPN league 95050 was correctly imported with real ESPN member GUIDs across 2011-2026.
- The e2e/screenshot fixture path reused provider league id `95050`, so fixture members like `member-01` and names like
  `Fixture Manager 01` were written into the same internal league namespace in the shared dev DB.
- The Record Book display rule uses most-recent team/person data. Fake 2026 fixture rows therefore buried valid
  historical real names.
- T2 improved provider-name refresh, but it did not hold because stale foreign rows could still coexist with provider
  truth and no invariant failed the contaminated league.

## General Fixes
- Fixture isolation:
  - `src/onboarding/fixture-espn.ts` now exports `FIXTURE_ESPN_PROVIDER_LEAGUE_ID = "fixture-espn-95050"`.
  - Mock ESPN fan/league fixture responses are cloned into that reserved non-real namespace.
  - `e2e/espn-onboarding.spec.ts` and `e2e/screenshots.spec.ts` delete the reserved fixture league after use.
  - `src/providers/espn/client.ts` still normalizes real numeric ESPN ids, while allowing string fixture ids in mock
    fan discovery.
- Import reconciliation:
  - `src/ingestion/current-league.ts` reconciles fetched seasons after upsert.
  - For each fetched `(league, season)`, members/teams absent from the fresh provider payload are removed for that
    season only.
  - `team_season` stale rows are removed directly, and stale `fantasy_teams` removal prevents derived rows from being
    recreated.
  - `src/ingestion/historical-import.ts` passes reconciliation seasons for each imported historical bundle and resolves
    identities afterward.
- Identity cleanup:
  - `resolveLeagueIdentities` now deletes orphan `identity_mapping` and orphan `persons` left after reconciliation.
  - Known placeholder canonical names are refreshed from provider owner names even if an older manual/steward row made
    them sticky.

## Integrity Invariant
- New `data_integrity_check_key` value: `provider_identity_contamination`.
- Real ESPN namespaces require member ids matching braced GUIDs: `{8-4-4-4-12}`.
- Known placeholders fail:
  - `Fixture Manager...`
  - `Screenshot ... Steward`
- Mixed real and invalid provider ids fail, as do mappings/team-owner names that still reference invalid ids or
  placeholder names.
- Reserved/test provider league ids such as `fixture-espn-95050` are intentionally skipped by real-id-format checks.
- `runDataIntegrityChecksInContext` replaces stale unresolved pass/fail snapshots for a check key/season on rerun,
  while preserving reviewed rows.

## Gotchas For Future Provider/Player-Depth Work
- Add provider-specific real-id-format rules before enabling a real provider namespace in the invariant.
- Keep all fixture provider league ids visibly non-real and reserved. Do not use plausible real ids in e2e or screenshot
  fixtures.
- Reconciliation is intentionally per season and per fetched data class. Never delete other seasons just because one
  season was imported.
- Provider credentials are still server-only; the verifier loads `.env.local` but never prints cookies.
