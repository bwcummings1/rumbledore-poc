# T7 Handoff - Edit Ledger / Change Log

## What T7 Built
- Added `/leagues/[leagueId]/ledger` as a separate league navigation destination between Data Book and Records.
- The route is read-only and member-visible. It uses the same page guard shape as Data Book/Records:
  `requireLeagueRole(minRole: "member")`, `markLeagueOpened`, then a server read model.
- The feed renders existing curation history only:
  - ordinary data edits from `league_data_edits`
  - checkpoint save markers (`targetKind="curation_checkpoint"`, `field="checkpoint_save"`)
  - season push markers (`targetKind="curation_push"`, `field="season_push"`)
- Rows are notification-style, newest-first, and expandable by keyboard-accessible buttons.
- Expanded rows show coral/jade before/after panels with non-color `[-] Before` / `[+] After` labels, plus field,
  scope, actor, timestamp, target, reason, source, and covered seasons where present.
- The existing steward public-ledger preview/drawer now reuses the same feed renderer.

## Key Decisions
- The standalone route filters the unified ledger to `source === "league_data_edit"` so the T7 feed stays focused on
  data edits, saves, and pushes instead of older identity/integrity audit rows.
- Actor display names are joined from central `users` when available; the immutable actor id is still shown in short
  form in the feed metadata.
- Saves and pushes derive their display details from the marker row JSON payloads written by T4. T7 does not query or
  mutate checkpoint/push tables directly.
- The route uses the league-feed masthead pattern and a bottom `TabLinks` row for the data-layer peer destinations:
  Data Book, Edit Ledger, Records.
- No save, restore, or push controls were added. The only write performed by T7 code is in the screenshot harness,
  where it seeds one edit/save/push so the screenshots are non-empty.

## What T8 Needs To Know
- T8 save/push controls should link back to `/leagues/[leagueId]/ledger` after successful actions, or include an
  "Open Edit Ledger" affordance near save/push confirmation states.
- T8 should continue to use existing mutation APIs:
  - save: `POST /api/leagues/[leagueId]/curation/checkpoints`
  - push season: `POST /api/leagues/[leagueId]/curation/push` with `action: "push"`
  - push all: `POST /api/leagues/[leagueId]/curation/push` with `action: "pushAll"`
- Those actions already write marker rows that this Ledger route will render automatically. No new ledger-specific
  mutation is needed.
- The shared feed component accepts `initialOpenIds` if T8 wants to deep-link or auto-open the newest row after a
  successful save/push.
