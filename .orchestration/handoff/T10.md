# T10 Handoff - Era Auto-Proposals

## What T10 Built
- Added a settings-signature era detector in `src/stats/curation.ts`.
  - Signals: league size, playoff matchup length, playoff team count, regular-season week count, and normalized lineup slot counts.
  - No owner/member churn detection; no regular/playoff segment proposals.
  - Single-format leagues return zero proposals.
- Persisted durable dismissed proposal state:
  - `league_season_grouping_status` now includes `dismissed`.
  - Migration: `src/db/migrations/0055_era_proposal_dismissed.sql`.
- Extended grouping service/API:
  - `proposeLeagueSeasonGroupings` now creates named/rationalized settings-derived proposals.
  - `confirmLeagueSeasonGrouping` rejects dismissed groupings.
  - `dismissLeagueSeasonGrouping` marks a proposed grouping dismissed and writes `grouping_dismissal` to `league_data_edits`.
  - `POST /api/leagues/[leagueId]/curation/groupings` now accepts `action: "confirm" | "dismiss"` and is `data_steward` gated.
- Added Data Book UI in the Settings grain:
  - Proposed/confirmed era cards show name, season range, rationale, and status.
  - Proposed cards expose Confirm, Adjust (name/seasons), and Dismiss.
  - Confirmed groupings still need Save + Push before Records sees them.
- Record Book code was not changed except tests; the existing pushed-snapshot lens now receives confirmed groupings once pushed.

## Gotchas
- Proposal creation is permission-aware in the Data Book loader: steward-capable viewers call `proposeLeagueSeasonGroupings`; ordinary members only list existing groupings.
- `proposeLeagueSeasonGroupings` intentionally returns existing groupings if any era grouping already exists, including dismissed/confirmed rows. This prevents re-spawning dismissed proposals on refresh.
- Dismissal is a status, not deletion, so ledger/filtering code should either ignore `dismissed` rows or display them intentionally.
- The real 95050 data has more roster-format granularity than the initial task examples: OP slot counts changed in 2015, hybrid OP/FLEX appears in 2016-2019, FLEX-only starts in 2020, and 14-week regular seasons start in 2021.
- Confirmed grouping changes are draft data until save/push. Tests and UI copy preserve that boundary.
