# T14 Handoff - Player-Depth League Data

## What landed

- ESPN player-depth import is now wired for current and historical seasons.
  - Current/history roster fetches request `mBoxscore`, `mMatchupScore`, `mScoreboard`, and `kona_player_info`.
  - Sparse historical box-score rows are merged with `mRoster` rows so weekly facts keep box-score points while roster rows supply lineup slot and starter/bench state.
  - Draft picks come from `mDraftDetail` plus `kona_player_info`.
  - Transactions use `mTransactions2` with the cwendt-compatible `transactions.filterType` header and a fallback to unfiltered reads.
- New/extended normalized model:
  - `NormalizedPlayer`
  - `NormalizedRosterEntry` with `actualPoints`, `projectedPoints`, `started`, and optional embedded player
  - `NormalizedDraftPick`
  - `NormalizedTransaction.scoringPeriod`
- New/extended DB model:
  - `fantasy_players` (league-scoped provider player identity, optional `nfl_player_id`)
  - `fantasy_roster_entries.fantasy_player_id`, `actual_points`, `projected_points`, `started`
  - `fantasy_draft_picks`
  - `fantasy_transactions.scoring_period`
  - Migration: `src/db/migrations/0058_player_depth.sql` and journal entry.
- Ingest/reconcile:
  - `persistNormalizedLeagueRows` now upserts fantasy players, roster entries, draft picks, and transactions.
  - Per-season reconciliation deletes stale roster/draft/transaction rows for fetched seasons and removes orphan fantasy players afterward.
  - Optional mapping to substrate B (`nfl_players.fantasy_provider_ids`) is best-effort only; B is still mock/$0 and never required.
- Data Book surface:
  - Weeks grain now shows a selected team-week roster panel with player, position, pro team, slot, starter/bench, actual, and projected points.

## Integrity

- Added `roster_coverage` and `player_points_rollup` integrity keys.
- `roster_coverage` requires roster rows for provider seasons with roster coverage.
- `player_points_rollup` compares starter player totals to team totals only for complete, single-scoring-period rows.
  - Rows with provider-missing starter points are recorded under `skippedTeamWeeks`.
  - Multi-week playoff rows are also skipped for rollup arithmetic because team totals span more than one scoring period.

## Important caveats

- ESPN 95050 returned **zero** transaction rows through `mTransactions2`, even with the transaction filter. Parser and persistence tests cover representative add/drop/trade payloads, and rows will persist when ESPN/provider payloads expose them.
- Older ESPN historical box-score payloads often omit weekly points for some starters. The importer keeps those roster rows, leaves point columns null, and the integrity check records skipped rollups instead of inventing totals.
- ESPN D/ST player ids can be negative; the parser now preserves real negative provider player ids and only rejects `0`/`-1` placeholders.
- Deeper player-level Record Book records, draft/transaction UI, and real substrate-B provider wiring are follow-ons.
