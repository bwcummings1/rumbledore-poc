## Schema Shape
All four tables are central/shared reference data. They have no `league_id`, no RLS policy, no edit ledger, no curation
checkpoint, and no push gate.

Common provenance columns:
- `source`
- `fetched_at`
- `content_hash`

Table grains:
- `nfl_players`: one player per source player id; includes `full_name`, `position`, `team`, and
  `fantasy_provider_ids` for provider-specific roster enrichment.
- `nfl_schedule`: one NFL game per source game id; includes season/week, teams, time, status, and scores.
- `nfl_team_stats`: one team box-score row per season/week/team.
- `nfl_player_week_stats`: one player-week fact row, linked to `nfl_players.id`.

## Ingest + Integrity
- Entry point: `ingestMockGeneralStats(db, { fixture?, fetchedAt? })`.
- Default source: `loadMockGeneralStatsFixture()`.
- Integrity entry point: `runGeneralStatsIntegrityChecks(fixture)`.
- Checks:
  - no silent empty fixture sections
  - unique players
  - unique games
  - every scheduled game has both team box rows and final scores match
  - player stat rows reference known players/games
  - player stat team/opponent align to the schedule
- The ingest throws `GeneralStatsIntegrityError` before DB writes if any check fails.
- Upserts are idempotent by stable keys plus `content_hash`; unchanged facts are not rewritten on a second ingest.

## Consumer API
Import from `@/general-stats`:
- `findGeneralStatsPlayerBySourceId(db, { source?, sourcePlayerId })`
- `findGeneralStatsPlayerByFantasyProviderId(db, { source?, provider, providerPlayerId })`
- `findGeneralStatsPlayersByName(db, { source?, name, limit? })`
- `getGeneralStatsPlayerStats(db, { source?, sourcePlayerId, season, week? })`
- `getGeneralStatsTeamBoxScore(db, { source?, team, season, week })`
- `getGeneralStatsSchedule(db, { source?, season, week?, team? })`
- `enrichLeagueRosterFactWithGeneralStats(db, fact, { source? })`

Empty/unpopulated substrate behavior is graceful: lookups return `null` or `[]`.

## Gotchas For Future News/AI Wiring
- Do not read the B tables directly from News/AI code unless the consumer API is missing a needed query. Extend
  `src/general-stats` first.
- B is shared reference data, not league-scoped data. Do not add league RLS, curation rows, or user edit flows.
- `fantasy_provider_ids` is the bridge from league roster facts to real NFL players. Prefer provider id enrichment over
  name matching; name matching is a fallback.
- The mock fixture uses real-looking provider ids, but it is still committed fixture data and must not be treated as a
  paid/live data feed.
