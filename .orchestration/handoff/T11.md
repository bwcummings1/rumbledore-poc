# T11 Handoff - Records Catalog Expansion

## What T11 Built
- Expanded the Record Book catalog in `src/stats/records-catalog.ts` into a typed category registry:
  - All-time
  - Regular season
  - Playoff
  - Head-to-head
  - Achievements
  - Lowlights
- Added new record outputs while keeping Records as a read-only projection over `composeCanonicalSnapshot` pushed data:
  - Playoff record leaders, playoff PF/PA, playoff win percentage, championships, runner-ups.
  - Regular-season titles.
  - H2H longest streak summaries in addition to rivalry ledgers.
  - Achievement records for high weeks/seasons, top scoring-week counts, win streaks, and titles.
  - Lowlight records for lowest single week, biggest and narrowest losses, longest losing streaks, worst career/season win percentage, lowest PF season, lowest season scoring average, most bottom scoring weeks, most last-place finishes, and playoff losses/PA.
- Updated the Records page to render those categories as page sections under the existing `PublicationMasthead` and `TabLinks` section-anchor pattern. The segment x confirmed-era lens remains a view control in the top card.
- No `.orchestration/legacy-records.md` or similar recovered legacy catalog file was present, so the shipped registry is the extension point.

## Gotchas
- The catalog receives lens-filtered pushed snapshot rows. It must not read draft/materialized live facts or define eras.
- Single-week records continue to exclude multi-week spans through the existing weekly-entry filter, and byes remain scoring-only with no W/L/H2H/streak effect.
- Loss-margin records use losing-side margins as positive display magnitudes.
- The display rule is unchanged: latest pushed team name plus real name, one row per person.
- `src/ai/personal-agent.test.ts` only changed because fixture builders need the expanded catalog shape.
